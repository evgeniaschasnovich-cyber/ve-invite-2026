## Цель проекта

Подготовить удобное интерактивное свадебное приглашение, которое корректно работает на телефонах и компьютерах, позволяет гостям получить всю необходимую информацию и подтвердить присутствие.

## Принцип взаимодействия

Владелец проекта не обязан разбираться в программировании и инфраструктуре. Технический исполнитель предлагает рекомендуемый безопасный вариант по умолчанию, объясняет решения и последствия простым языком и не перегружает владельца лишними техническими деталями. Значимые решения — публикация, расходы, доступы, приватность гостей и выбор внешних сервисов — принимаются только после понятного согласования с владельцем проекта.

## Решения, которые нужно принять

1. Куда должны приходить ответы RSVP?
2. Нужна ли гостям возможность указывать сопровождающего и количество гостей?
3. Какая ссылка будет использоваться для дресс-кода/вдохновения?
4. Нужны ли фотографии, музыка, галерея и анимации?
5. Нужен ли собственный домен?
6. Нужна ли приватность: скрытая ссылка, пароль или ограничение доступа?

## Принятые решения

- Для публикации используется GitHub Pages.
- Сайт может быть доступен по открытой ссылке, но не должен предлагаться поисковым системам для индексации.
- В HTML и `robots.txt` добавляются инструкции `noindex`/`Disallow`; они снижают вероятность обнаружения, но не являются защитой доступа.
- Сайт планируется удалить после мероприятия 7 августа 2026 года.
- До появления отдельной ссылки для вдохновения на сайте используется встроенная подборка подсказок по образам.
- Три исходных изображения приглашения используются как визуальный источник истины для композиции и текстов.
- Ответы RSVP собираются через Google Forms и сохраняются в Google Sheets; собственный сервер и Railway для этого не используются.
- Свободный выбор количества гостей не используется; необязательное поле имени сопровождающего заполняется только для приглашений на двоих.
